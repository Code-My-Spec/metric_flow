## Issues

### Cancel link navigates to / instead of /dashboards (stale server)

#### Severity
LOW

#### Scope
APP

#### Description
The Cancel link on both `/dashboards/new` and `/dashboards/:id/edit` navigates to `/` (the home/All Metrics page) instead of `/dashboards` (the dashboard list).

The source code fix (`navigate="/dashboards"`) is present in `lib/metric_flow_web/live/dashboard_live/editor.ex` line 67, but the currently running server is serving old compiled code that renders href `/`. Per the task prompt, this is a known stale server issue — the fix has been made and will take effect after a server restart.

To verify: navigate to `http://localhost:4070/dashboards/new` and inspect the Cancel button href. Current observed value: `/`. Expected after server restart: `/dashboards`.
