### Scenario 3 — Clicking Sync Now triggers sync and shows loading state (criteria 4051, 4052)

fail

Clicked the "Sync Now" button on the Google card. Instead of showing "Sync started for Google" and entering a loading/syncing state, the LiveView showed an error flash: **"Integration not found."** The button remained enabled (never disabled). No "Syncing" badge appeared. No `.badge-warning` with text "Syncing" was found inside `[data-role="integration-sync-status"]`.

Root cause identified: `DataSync.sync_integration/2` calls `SyncJobRepository.create_sync_job(scope, integration.id, %{provider: :google})`. The `SyncJob` schema defines `@providers [:google_analytics, :google_ads, :facebook_ads, :quickbooks]` — the `:google` atom is not a valid enum value. The `SyncJob.changeset/2` rejects it, causing `create_sync_job` to return `{:error, changeset}`. The `with` chain in `sync_integration/2` propagates this error, and the `handle_event("sync", ...)` clause catches it with `{:error, _reason}` and shows "Integration not found." — a misleading error message for a schema validation failure.

No Oban job was enqueued. The `sync_jobs` table remained empty.

Evidence: `.code_my_spec/qa/438/screenshots/03-after-sync-click.png`, `.code_my_spec/qa/438/screenshots/03-sync-error-state.png`

## Issues

### SyncJob schema rejects :google provider — create_sync_job fails silently as "Integration not found"

#### Severity
HIGH

#### Scope
APP

#### Description

`MetricFlow.DataSync.SyncJob` defines its provider enum as `@providers [:google_analytics, :google_ads, :facebook_ads, :quickbooks]` (line 39 of `lib/metric_flow/data_sync/sync_job.ex`). The `:google` atom is absent. When a user clicks "Sync Now" for their Google integration:

1. `DataSync.sync_integration(scope, :google)` calls `SyncJobRepository.create_sync_job(scope, integration.id, %{provider: :google})`
2. `SyncJob.changeset/2` casts `provider: :google` against the Ecto.Enum — `:google` is not a valid value so the cast is rejected (field becomes invalid)
3. `create_sync_job` returns `{:error, %Ecto.Changeset{}}` instead of `{:ok, sync_job}`
4. The `with` chain in `sync_integration/2` propagates `{:error, changeset}`
5. `handle_event("sync", ...)` catches `{:error, _reason}` and shows "Integration not found." — incorrect error message
6. No Oban job is enqueued, no sync is performed, no loading state is shown

Confirmed by: empty `sync_jobs` table after clicking Sync Now, and the error flash appears immediately (not after a server-side delay).

Reproduction: Log in as `qa@example.com`, navigate to `http://localhost:4070/integrations`, click "Sync Now" on the Google card. Expected: "Sync started for Google" flash and loading state. Actual: "Integration not found." flash, button stays enabled.

Fix: Add `:google` to `@providers` in `lib/metric_flow/data_sync/sync_job.ex`, and generate a migration to update the column constraint if necessary. Also update `SyncWorker.providers_for/1` to ensure `:google` is handled (it already has a clause: `def providers_for(:google), do: {:ok, [GoogleAnalytics, GoogleAds]}`).

Also consider improving the error message in `handle_event("sync", ...)` to distinguish between "integration not found in DB" vs "sync job creation failed" vs "unsupported provider" for better user feedback.
