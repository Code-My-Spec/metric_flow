## Issues

### Smart mode does not implement top-positive and top-negative correlation sections

#### Severity
HIGH

#### Description
The BDD spec (criterion 4125) requires that Smart mode display two subsections:
- `[data-role='top-positive-correlations']` containing up to 5 `[data-role='correlation-row']` elements (highest positive coefficients)
- `[data-role='top-negative-correlations']` containing up to 5 `[data-role='correlation-row']` elements (strongest negative coefficients)

The current implementation in `lib/metric_flow_web/live/correlation_live/index.ex` renders only a generic AI recommendations card and a feedback panel within Smart mode. Neither the `top-positive-correlations` nor the `top-negative-correlations` data-role attributes exist anywhere in the template. A search of the full page HTML confirmed their absence.

The feature is particularly notable because the QA account has real correlation data with both positive coefficients (e.g., Google Ads clicks at 0.82, spend at 0.74) and negative coefficients (e.g., QuickBooks income at -0.38). Smart mode should surface this segmented view but instead shows only generic recommendation text.

Reproduction steps:
1. Log in as qa@example.com
2. Navigate to `http://localhost:4070/correlations`
3. Click the Smart button in the mode toggle
4. Click "Enable AI Suggestions"
5. Observe: no `[data-role='top-positive-correlations']` or `[data-role='top-negative-correlations']` sections are rendered
