### Scenario 3: Sync Now on expired integration shows reconnection message

fail (code review finding)

Browser testing was blocked by missing Vibium MCP tools, but a direct code audit reveals the acceptance criterion is not met.

In `lib/metric_flow_web/live/integration_live/index.ex` at line 300:

```elixir
{:error, reason} when reason in [:not_found, :not_connected] ->
  {:noreply, put_flash(socket, :error, "Integration not found.")}
```

When `DataSync.sync_integration/2` returns `{:error, :not_connected}` (which happens when the integration's token is expired and no refresh token is available), the LiveView flash message is `"Integration not found."` — not the expected "token has expired. Please reconnect." message.

The `DataSync.sync_integration/2` contract (in `lib/metric_flow/data_sync.ex`) explicitly documents that `:not_connected` means "integration exists but its token is expired and no refresh token is available" — this is a distinct case from `:not_found`. The LiveView handler conflates both cases, showing a misleading "not found" message rather than the reconnection prompt the acceptance criteria require.

The `SyncWorker.format_error(:token_expired)` function (`lib/metric_flow/data_sync/sync_worker.ex`, line 432) does produce the correct message `"Token expired and could not be refreshed. Please reconnect."` — but this code path is never reached because `check_connected/1` in `DataSync` guards against dispatching the Oban job at all when the token is expired. The error surface is at the LiveView handler level, not the worker level.

## Issues

### Sync Now on expired integration shows wrong flash message

#### Severity
HIGH

#### Description
When a user clicks "Sync Now" on a Google Analytics integration with an expired token (no refresh token), the LiveView shows the flash error `"Integration not found."` instead of the expected `"token has expired. Please reconnect."` message.

Root cause: `IntegrationLive.Index.handle_event("sync", ...)` in `lib/metric_flow_web/live/integration_live/index.ex` (lines 300–302) handles both `:not_found` and `:not_connected` with the same flash message:

```elixir
{:error, reason} when reason in [:not_found, :not_connected] ->
  {:noreply, put_flash(socket, :error, "Integration not found.")}
```

The fix is to handle `:not_connected` separately with a message directing the user to reconnect, such as: `"Your Google Analytics token has expired. Please reconnect."` The `DataSync.SyncWorker.format_error(:token_expired)` function already has the right message template — it just needs to be wired into the LiveView handler for the pre-dispatch `:not_connected` case.

Reproduction steps:
1. Seed an expired `google_analytics` integration: `mix run priv/repo/qa_seeds_440.exs`
2. Log in as `qa@example.com` and navigate to `/integrations`
3. Click "Sync Now" on the Google Analytics card
4. Observe flash message — shows "Integration not found." instead of reconnection prompt
