## Issues

### "View in Visualizations" link navigates to /dashboards instead of /visualizations

#### Severity
MEDIUM

#### Description
In the save confirmation section (`[data-role="save-confirmation"]`), the link labeled "View in Visualizations" uses `navigate="/dashboards"` and routes to `http://localhost:4070/dashboards` (the Dashboards page). The correct destination should be `http://localhost:4070/visualizations` — a distinct route that exists in the router (`live "/visualizations", VisualizationLive.Index, :index`).

Source location: `lib/metric_flow_web/live/ai_live/report_generator.ex` line 136.

The current code is:
```heex
<.link navigate="/dashboards" class="link block mb-3">
  View in Visualizations
</.link>
```

It should be:
```heex
<.link navigate="/visualizations" class="link block mb-3">
  View in Visualizations
</.link>
```

Reproduced consistently after any successful save on `http://localhost:4070/reports/generate`.

### Chart renders with empty data (no bars visible)

#### Severity
INFO

#### Description
The AI-generated Vega-Lite spec produces a valid chart structure (title, axes, correct chart type) but the embedded data values are zero/empty, resulting in a chart with labeled axes and no visible bars or data points. This is expected behavior since the feature generates illustrative specs without a live data source, but users may find the empty chart confusing. The chart is still structurally correct and demonstrable.

Observed on: `http://localhost:4070/reports/generate` after submitting "Show me a bar chart of monthly revenue for the past 6 months".
