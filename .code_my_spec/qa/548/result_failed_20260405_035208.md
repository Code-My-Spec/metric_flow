## Issues

### Webhook route not added to router

#### Severity

HIGH

#### Scope

APP

#### Description

The `BillingWebhookController` is implemented but the route `POST /billing/webhooks` has not been added to `lib/metric_flow_web/router.ex`. A route like the following is needed under the `:api` pipeline:

```elixir
scope "/billing", MetricFlowWeb do
  pipe_through :api
  post "/webhooks", BillingWebhookController, :handle
end
```

### No subscription state persistence

#### Severity

MEDIUM

#### Scope

APP

#### Description

The Billing context currently logs events but does not persist subscription state changes to the database. The `Subscription` schema, `Plan` schema, and `BillingRepository` have not been created yet. Webhook events are acknowledged but have no lasting effect.
