## Issues

### No top-level /dashboard route — BDD specs reference /dashboard but spec says no-param route

#### Severity
INFO

#### Description
The brief flagged a route discrepancy, but this is resolved: the router has `live "/dashboard", DashboardLive.Show, :index` which is exactly what the spec describes. The `/dashboards/:id` route is an additional alias. No routing issue exists. The previously filed issue `qa-441-no_top_level_dashboard_route_bdd_specs_re.md` appears to have been based on incorrect information — the `/dashboard` route is present and functional.

### Onboarding state untestable — QA account has integrations connected

#### Severity
LOW

#### Scope
QA

#### Description
Scenario 3 (onboarding prompt when no integrations connected) could not be tested because the QA seed account has integration data in the database. The `Dashboards.has_integrations?/1` check returns `true` for `qa@example.com`, so the onboarding prompt is never rendered.

To test the onboarding state, the QA seed script should either: (a) include a separate user/account with no integrations, or (b) provide a teardown step that removes integrations for the QA account. Alternatively, a freshly registered test account (no integrations) could be used.

### Google Analytics metrics classified as platform-specific with "Unknown" provider

#### Severity
MEDIUM

#### Description
The dashboard shows Google Analytics metrics (activeUsers, averageSessionDuration, bounceRate, newUsers, screenPageViews, sessions) in the Platform-Specific Metrics section, each labelled "Unknown only" as the provider attribution. These metrics are not in the canonical taxonomy (`@known_raw_metrics`: Clicks, Spend, Impressions, Revenue, Conversions), so they appear as platform-specific. However, the provider field shows "Unknown" rather than "Google Analytics" or the integration's provider name.

Reproduced at `http://localhost:4070/dashboard` with `qa@example.com`. The `stat[:provider]` value is `nil` for these records, causing the fallback "Unknown" label in `data-role="platform-source"`.
