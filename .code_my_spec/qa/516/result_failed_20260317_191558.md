## Issues

### sync_history error_message field causes DB truncation error displayed in UI

#### Severity
HIGH

#### Description
The sync history page displays raw database errors in the UI. Several entries show:

```
ERROR 22001 (string_data_right_truncation) value too long for type character varying(255)
```

This indicates the `error_message` column in the `sync_history` table has a `character varying(255)` constraint. When a sync fails with an error message longer than 255 characters, Ecto attempts to insert the full error string and receives `ERROR 22001` from PostgreSQL. This truncation error is then stored as the error message in a subsequent save, or the raw DB error is bubbled up and displayed in the UI.

Observed at `http://localhost:4070/integrations/sync-history` — multiple "Google" failed entries show the raw DB truncation error instead of the actual sync failure reason. The actual cause of the sync failure (e.g., missing GA4 property, missing Google Ads customer ID) is lost.

Reproduction: Trigger a sync that produces an error message longer than 255 characters. The `[data-role="sync-error"]` element will display the DB truncation error instead of the actual failure reason.

### Scenario 3 empty state not verifiable — seeds not re-run due to server conflict

#### Severity
LOW

#### Scope
QA

#### Description
Scenario 3 requires no sync history records in the database. The brief states "seeds clear all sync history records" and recommends skipping seeds if login works. However, if prior QA runs have accumulated sync history, the empty state cannot be tested without re-running seeds.

Re-running seeds while the Phoenix server is running fails due to the Cloudflare tunnel GenServer conflict. The workaround (`mix run --no-start -e "..."`) is documented in the QA plan but was not applied here.

To test Scenario 3, run:
```
mix run --no-start -e "Application.ensure_all_started(:postgrex); Application.ensure_all_started(:ecto); MetricFlow.Repo.start_link([])" priv/repo/qa_seeds.exs
```
This clears the sync history and allows the empty state to be verified.
