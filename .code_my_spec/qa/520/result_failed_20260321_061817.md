## Issues

### Integrations index does not display selected Google Business locations

#### Severity
MEDIUM

#### Description
The integrations index page (`/integrations`) shows "No accounts selected" for Google Business even when `included_locations` has been saved to the integration's `provider_metadata`. The root cause is in `index.ex`: `metadata_key_for_provider/1` (lines 498–503) has no `:google_business` clause and falls through to the catch-all `"property_id"`. `selected_account_display/1` then looks up `"property_id"` in the metadata, which is absent, and returns `nil`. The function also has no branch to handle an array value from `"included_locations"`.

Fix: add `defp metadata_key_for_provider(:google_business), do: "included_locations"` to the `index.ex` function, and update `selected_account_display/1` to handle the list case — e.g., `Enum.join(value, ", ")` when `is_list(value)`.

### Missing-location flagging requires live GBP API (criterion 4862 blocked)

#### Severity
HIGH

#### Description
When a previously configured location ID is no longer available (e.g., a deleted or revoked location), the accounts page shows no warning. The manual entry field is pre-filled with the stale location ID with no UI feedback.

The `compute_missing_locations/3` function (connect.ex lines 748–757) only computes missing locations when `fetched_accounts != []`. Since the live GBP API requires valid OAuth credentials, a test token produces no API results, so the comparison never runs and the warning never appears.

This is a partial implementation gap: the template already renders `[data-role='missing-location']` and `[data-role='location-unavailable']` (lines 292–304 of connect.ex), but it can only trigger when the live API returns location data. In QA, this scenario cannot be fully validated without real GBP API access (valid OAuth token and a real GBP account with locations).

Reproduced by setting `included_locations` to `["accounts/123/locations/deleted-loc-1"]` in the integration's `provider_metadata` and navigating to `/integrations/connect/google_business/accounts`.
