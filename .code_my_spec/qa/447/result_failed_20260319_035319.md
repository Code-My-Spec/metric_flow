## Issues

### No-data empty state untestable: qa-member shares the team account's correlation data

#### Severity
MEDIUM

#### Scope
QA

#### Description
The brief states: "Log in as `qa-member@example.com` (no correlation data)" to test the no-data empty state. However, both `qa@example.com` and `qa-member@example.com` belong to "QA Test Account", which has real correlation results from connected integrations. The `/correlations` LiveView resolves scope to the team account, so `qa-member@example.com` sees the same 27 correlation rows as the owner.

The `qa_seeds_447.exs` script creates data on `qa@example.com`'s personal account, but at runtime the active scope is the team account. The no-data empty state (`[data-role="no-data-state"]`) was never rendered during this QA session.

To test the no-data state, a dedicated isolated account with no integrations and no correlation results is required, or the test environment must be configured to use the personal account scope.

### Insufficient-data-warning badge not shown when account has existing correlation data

#### Severity
MEDIUM

#### Scope
APP

#### Description
When clicking "Run Now" returns `{:error, :insufficient_data}`, the LiveView sets `@run_error = :insufficient_data` and shows a flash error message. The source also renders `[data-role="insufficient-data-warning"]` badge, but this element is nested inside `[data-role="no-data-state"]`, which only renders when `@summary.no_data` is true.

In practice, if an account has existing correlation results (any prior run), `@summary.no_data` is false — the data-state view is shown. When the user clicks Run Now and gets an `:insufficient_data` error, the flash appears but the badge does not, because its container `[data-role="no-data-state"]` is hidden.

To reproduce:
1. Log in to an account that has prior correlation results but whose underlying metric data is insufficient for a new run
2. Click Run Now
3. Flash error: "Not enough data to run correlations. At least 30 days of metric data is required." appears
4. `[data-role="insufficient-data-warning"]` does not appear (it is inside the hidden `no-data-state` div)

The badge should be conditionally rendered in the raw-mode or summary area so it appears regardless of whether the no-data empty state is shown.

### Data points shows "0 data points" in correlation summary bar

#### Severity
LOW

#### Scope
APP

#### Description
The summary bar's `[data-role="data-points"]` shows "0 data points" for the QA Test Account even though 27 correlation result rows are visible in the table. The team account's completed `CorrelationJob` record has `data_points: 0` while the actual results exist. This is a data consistency issue — the job's `data_points` field is not updated to reflect the actual number of results or metric data points used in the calculation. Users see "0 data points" which is misleading when results are clearly present.

Reproduced at: `http://localhost:4070/correlations` logged in as `qa@example.com` (QA Test Account active).
