## Issues

### B9 brief incorrectly specifies qa-member@example.com for empty-state testing

#### Severity
LOW

#### Scope
QA

#### Description
The brief (Scenario B9) instructs the tester to log in as `qa-member@example.com` to test the empty insights state. However, `qa-member@example.com` is a member of "QA Test Account" which has 5 seeded insights. When this user visits `/insights`, they see all 5 cards — not the empty state.

The correct user for empty-state testing is `qa-empty@example.com`, as documented in `qa_seeds_450.exs`: "This user has their own isolated personal account with NO insights." The brief should be updated to reference `qa-empty@example.com` for Scenario B9.

### B10 empty filter state not testable with current seed data

#### Severity
INFO

#### Scope
QA

#### Description
Scenario B10 requires at least one suggestion type filter to match zero insights. The seed data in `qa_seeds_450.exs` creates exactly one insight for each of the five suggestion types (budget_increase, optimization, monitoring, budget_decrease, general), so every filter button returns at least one result. The `[data-role='no-filter-results-state']` empty filter state UI was never rendered and could not be validated.

To make B10 testable, `qa_seeds_450.exs` should omit at least one suggestion type, or the brief should note that this scenario cannot be exercised with the current seed data.

### Dashboard AI info buttons not implemented (AC3/AC4 gap)

#### Severity
LOW

#### Description
The acceptance criteria for this story include "Each chart can have an AI info button" and "Clicking AI button shows insights". No `[data-role='ai-info-button']` elements were found anywhere on the dashboard at `http://localhost:4070/dashboard`, including after scrolling through the full page. The dashboard renders charts and a data table but has no AI info entry points. The brief notes this as a potential implementation gap.
