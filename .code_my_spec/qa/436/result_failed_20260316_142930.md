## Issues

### Disconnect flash message missing "no new data will sync" phrase

#### Severity
LOW

#### Scope
APP

#### Description

After confirming a disconnect, the flash message reads "Disconnected from Google. Historical data is retained." The spec (index.spec.md) describes the flash as "Disconnected from {platform name}. Historical data is retained; no new data will sync after disconnecting." and the brief (Scenario 12) also expects the additional "No new data will sync after disconnecting." phrase. The message is truncated — the second sentence is missing. Reproduced at `http://localhost:4070/integrations` — click Disconnect on any connected platform, then confirm. Source: `lib/metric_flow_web/live/integration_live/index.ex` line 327 — `"Disconnected from #{platform_display_name(socket.assigns.platforms, provider)}. Historical data is retained."`.

### Brief uses google_ads provider that does not exist

#### Severity
LOW

#### Scope
QA

#### Description

The brief (and several scenarios) reference `google_ads` as the provider name and expect platform name "Google Ads". The actual registered providers in the app are `:google`, `:facebook_ads`, and `:quickbooks`. There is no `:google_ads` provider. The seed script in `qa_seeds.exs` correctly uses `:google`. The brief scenarios for Scenario 3 (platform name), Scenario 7 (detail link), and Scenario 12 (flash message) should reference "Google" / `:google` instead of "Google Ads" / `:google_ads`. This was previously noted in accepted issue qa-436-google_ads_is_not_a_registered_provider_s.

### mix run seed script works without --no-start flag

#### Severity
INFO

#### Scope
QA

#### Description

The QA plan warns that `mix run priv/repo/qa_seeds.exs` will fail when the Phoenix server is running due to a Cloudflare tunnel GenServer conflict. In this QA run, `mix run priv/repo/qa_seeds.exs` ran successfully without the `--no-start` workaround. The Cloudflare tunnel is disabled in `config/dev.exs` (`enabled: false`), so no conflict occurs. The QA plan note about the Cloudflare tunnel conflict may no longer apply since the tunnel is disabled. The plan should be updated to reflect this.
