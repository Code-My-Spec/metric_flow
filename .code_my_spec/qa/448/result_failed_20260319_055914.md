## Issues

### QA seeds do not reset correlation state between runs

#### Severity
MEDIUM

#### Scope
QA

#### Description
The QA account (`qa@example.com`) retains correlation records from prior test runs. Scenarios 4 and 8 require an account with no correlation history (`summary.no_data == true`) to reach the no-data empty state and the insufficient-data error path respectively. Because the seeds script does not delete existing correlation records, these scenarios cannot be tested on a re-used QA account.

To make these scenarios fully testable, the seed script (`priv/repo/qa_seeds.exs`) should delete existing `correlation_results`, `correlation_jobs`, and related records for the QA account before inserting fresh seed data. Alternatively, the brief should note that Scenarios 4 and 8 require a freshly migrated database or a separate clean test account.

Reproduced consistently — both Scenarios 4 and 8 ended in partial coverage due to existing correlation data on the QA account.
