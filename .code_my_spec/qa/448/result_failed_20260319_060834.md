## Issues

### Scenario 4 and 8 cannot be tested: existing correlation summary in QA account

#### Severity
LOW

#### Scope
QA

#### Description
The QA account (`qa@example.com`) has a persistent correlation summary record in the database from prior QA runs. This causes `summary.no_data == false` on page load, so the `[data-role="no-data-state"]` empty state (Scenario 4) is never rendered and the Run Now button produces `{:ok, _job}` instead of `{:error, :insufficient_data}` (Scenario 8).

To test Scenario 4 (no-data empty state) and Scenario 8 (insufficient data error), the correlation summary record for the QA account would need to be deleted from the database before each test run, or a dedicated test account with no correlation history should be created.

Recommended fix: add a QA seed step that deletes existing `correlation_results` and `correlation_jobs` records for the QA account, or add a second test user (e.g. `qa-fresh@example.com`) with no correlation history for testing no-data paths.
