# QA Result: Agency Customer Billing (Story 547)

## Status

fail

## Scenarios

### 1. Agency user association via invite (criterion 5013)

**Status:** fail

No agency_id field exists on the User schema. Agency invite links do not associate users with agencies. The invitation system exists but does not track agency affiliation for billing purposes.

### 2. Agency plans shown on checkout (criterion 5014)

**Status:** fail

The checkout page is a stub showing only "MetricFlow Pro" as a hardcoded plan. No logic exists to detect agency affiliation and display agency-specific plans.

### 3. Checkout uses agency Stripe account (criterion 5015)

**Status:** fail

The subscribe button exists but has no event handler to create a Stripe Checkout session. No agency routing logic exists.

### 4. Successful payment stores subscription (criterion 5016)

**Status:** fail

No checkout success callback route exists. The checkout LiveView doesn't handle success/cancel return URLs.

### 5. Agency webhook events routed correctly (criterion 5017)

**Status:** partial

The `/billing/webhooks` endpoint exists and processes subscription events with 200 responses. However, there is no agency-specific routing — events are processed the same regardless of source Stripe account.

### 6. Subscription status synced via webhooks (criterion 5018)

**Status:** partial

Webhook processing updates subscription status in the database. However, no distinction is made between direct and agency subscriptions.

### 7. Disconnected agency preserves subscriptions (criterion 5019)

**Status:** fail

No logic exists to detect disconnected agency Stripe accounts or flag affected subscriptions.

### 8. Billing context immutable after subscription (criterion 5020)

**Status:** fail

No billing context tracking exists on subscriptions to distinguish direct vs agency billing.

## Issues

### Checkout LiveView is a stub with no Stripe integration

#### Severity

HIGH

#### Scope

APP

#### Description

The `MetricFlowWeb.SubscriptionLive.Checkout` LiveView displays a hardcoded plan card but has no event handlers, no Stripe Checkout session creation, no success/cancel handling, and no agency plan detection. Needs full implementation to support both direct and agency billing flows.
