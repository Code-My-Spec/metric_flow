### Scenario 8: BDD spec route /dashboards/new — route existence check

**Result:** fail (brief Setup Notes were incorrect)

The brief stated that `/dashboards/new` does not exist and the BDD spec targets a non-existent route. This is incorrect. The route `/dashboards/new` renders a Dashboard editor with heading "New Dashboard". Furthermore:

- `[data-role='add-visualization-btn']` exists on the page — the BDD spec's interaction model is valid
- Clicking "+ Add Visualization" reveals `[data-role='chart-type-selector']` with Line, Bar, and Area buttons
- The BDD spec flow (`/dashboards/new` → click `add-visualization-btn` → interact with `chart-type-selector`) matches the actual implementation

The brief's claim that the route does not exist and that the BDD spec has a "route mismatch" is false. Both `/visualizations/new` and `/dashboards/new` expose chart type selection, but the BDD spec correctly targets the Dashboard flow.

Evidence: `.code_my_spec/qa/442/screenshots/scenario_8_dashboards_new.png`
Evidence: `.code_my_spec/qa/442/screenshots/scenario_8_dashboards_new_after_add.png`

## Issues

### Brief incorrectly states /dashboards/new does not exist

#### Severity
MEDIUM

#### Scope
QA

#### Description
The brief's Setup Notes section claims "/dashboards/new" is a non-existent route and that the BDD spec has a "route mismatch." In reality, `GET /dashboards/new` renders a fully functional Dashboard editor LiveView. The `[data-role='add-visualization-btn']` button referenced in the BDD spec is present on this page, and clicking it correctly reveals `[data-role='chart-type-selector']` with Line/Bar/Area buttons. The BDD spec flow is valid. The brief redirected testing entirely to `/visualizations/new`, bypassing the actual BDD spec target and rendering Scenario 8's premise false.

Reproduction: Navigate to `http://localhost:4070/dashboards/new` while authenticated. Observe the "New Dashboard" editor with the "+ Add Visualization" button. Click it to reveal the chart type selector.
