## Issues

### Missing database migration for billing tables

#### Severity

HIGH

#### Scope

APP

#### Description

The `billing_plans`, `billing_subscriptions`, and `billing_stripe_accounts` tables referenced by the Billing schemas do not have Ecto migrations. The schemas (`MetricFlow.Billing.Plan`, `MetricFlow.Billing.Subscription`, `MetricFlow.Billing.StripeAccount`) exist but cannot persist data. A migration needs to be created with `mix ecto.gen.migration create_billing_tables`.

### StripeClient lacks product/price creation

#### Severity

MEDIUM

#### Scope

APP

#### Description

`MetricFlow.Billing.StripeClient` currently only implements webhook signature verification. It does not implement `create_product/2`, `create_price/3`, or `deactivate_price/1` which are needed for plan creation to sync with Stripe. Plans can be created locally but won't have Stripe Price IDs.
