# MetricFlow.Ai.Insight

Ecto schema storing an AI-generated insight derived from correlation analysis. Each record represents
one actionable recommendation produced by InsightsGenerator for a given account, optionally linked
to the specific CorrelationResult that motivated it. Insights carry a suggestion_type enum that
categorises the recommended action, a confidence score between 0.0 and 1.0, and a metadata map for
supplementary context such as metric names and correlation values.

schema

## Fields

| Field                  | Type              | Required   | Description                                                                   | Constraints                                       |
| ---------------------- | ----------------- | ---------- | ----------------------------------------------------------------------------- | ------------------------------------------------- |
| id                     | integer           | Yes (auto) | Primary key                                                                   | Auto-generated                                    |
| account_id             | integer           | Yes        | Foreign key to the owning account                                             | References accounts.id                            |
| correlation_result_id  | integer           | No         | Optional foreign key to the correlation result that motivated this insight    | References correlation_results.id, nullable       |
| content                | string            | Yes        | Full text of the AI-generated recommendation                                  | Min length: 1                                     |
| summary                | string            | Yes        | Short human-readable headline for the insight                                 | Max length: 500                                   |
| suggestion_type        | enum (atom)       | Yes        | Category of the recommended action; one of the five suggestion type values    | Values: budget_increase, budget_decrease, optimization, monitoring, general |
| confidence             | float             | Yes        | Model confidence score for this insight                                       | Between 0.0 and 1.0 (inclusive)                   |
| metadata               | map               | No         | Supplementary context, e.g. metric names and correlation coefficients         | Defaults to %{}                                   |
| generated_at           | utc_datetime_usec | Yes        | Timestamp when the insight was generated by the AI model                      |                                                   |
| inserted_at            | utc_datetime_usec | Yes (auto) | Record creation timestamp                                                     | Auto-generated                                    |
| updated_at             | utc_datetime_usec | Yes (auto) | Record last-update timestamp                                                  | Auto-generated                                    |

### suggestion_type values

| Value           | Meaning                                                                     |
| --------------- | --------------------------------------------------------------------------- |
| budget_increase | Recommend a direct spend increase on a platform or campaign                 |
| budget_decrease | Recommend a direct spend decrease on a platform or campaign                 |
| optimization    | Recommend a configuration or targeting change with no spend change          |
| monitoring      | Recommend continued observation with no immediate action required           |
| general         | Informational insight that does not correspond to a specific action         |

High-confidence threshold: confidence >= 0.7. Insights with confidence below this threshold are
considered low-confidence and may be surfaced with reduced emphasis in the UI.

## Functions

### changeset/2

Builds and validates an Ecto changeset for creating or updating an Insight record.

```elixir
@spec changeset(t(), map()) :: Ecto.Changeset.t()
```

**Process**:
1. Cast the following fields from attrs: account_id, correlation_result_id, content, summary, suggestion_type, confidence, metadata, generated_at
2. Validate that account_id, content, summary, suggestion_type, confidence, and generated_at are present
3. Validate content has a minimum length of 1 (empty strings are rejected)
4. Validate summary has a maximum length of 500 characters
5. Validate confidence is between 0.0 and 1.0 inclusive
6. Add an association constraint requiring the account record to exist
7. If correlation_result_id is present and non-nil, add an association constraint requiring the correlation_result record to exist

**Test Assertions**:
- returns a valid changeset when all required fields are present and valid
- returns an invalid changeset when account_id is missing
- returns an invalid changeset when content is missing
- returns an invalid changeset when summary is missing
- returns an invalid changeset when suggestion_type is missing
- returns an invalid changeset when confidence is missing
- returns an invalid changeset when generated_at is missing
- returns an invalid changeset when content is an empty string
- returns an invalid changeset when summary exceeds 500 characters
- returns a valid changeset when summary is exactly 500 characters
- returns an invalid changeset when confidence is below 0.0
- returns an invalid changeset when confidence is above 1.0
- returns a valid changeset when confidence is exactly 0.0
- returns a valid changeset when confidence is exactly 1.0
- returns a valid changeset when confidence is exactly 0.7 (boundary of high-confidence threshold)
- returns a valid changeset for each of the five suggestion_type enum values
- returns an invalid changeset for an unrecognised suggestion_type value
- returns a valid changeset when correlation_result_id is nil (association is optional)
- returns a valid changeset when a valid correlation_result_id is provided
- defaults metadata to an empty map when the field is not supplied in attrs
- database insert fails with an association error on account when account_id does not reference an existing account
- database insert fails with an association error on correlation_result when correlation_result_id does not reference an existing record

### actionable?/1

Returns whether the insight represents an actionable recommendation that the user should act on,
as opposed to an informational or monitoring note.

```elixir
@spec actionable?(t()) :: boolean()
```

**Process**:
1. Return false when suggestion_type is :monitoring
2. Return false when suggestion_type is :general
3. Return true for all other suggestion_type values (:budget_increase, :budget_decrease, :optimization)

**Test Assertions**:
- returns true for suggestion_type :budget_increase
- returns true for suggestion_type :budget_decrease
- returns true for suggestion_type :optimization
- returns false for suggestion_type :monitoring
- returns false for suggestion_type :general

### high_confidence?/1

Returns whether the insight's confidence score meets or exceeds the high-confidence threshold of 0.7.

```elixir
@spec high_confidence?(t()) :: boolean()
```

**Process**:
1. Compare the confidence field against the module-level threshold of 0.7
2. Return true if confidence >= 0.7, false otherwise

**Test Assertions**:
- returns true for confidence of 1.0
- returns true for confidence of 0.75
- returns true for confidence of 0.7 (boundary value, inclusive)
- returns false for confidence of 0.69 (just below boundary)
- returns false for confidence of 0.5
- returns false for confidence of 0.0
